IndentedLoggerAdapter API¶

API Reference for the IndentedLoggerAdapter

class python_log_indenter.IndentedLoggerAdapter(logger, extra=None, auto_add=True, **kwargs)¶

Overrides the init and parameters from logging.Logger, however adds it adds some functions.

Parameters:
  • auto_add (bool) – this controls how the indent is handled, if this is True (default) the indent is automatically added to the beginning of the message before it is passed to the logging engine. If False, the indent is passed as part of the logging.LogRecord as the parameter .indent_str. this allows access to the indent by custom formatting commands if needed.
  • spaces (int) – Sets the number of spaces for each indent
  • indent_char (str) – Sets the character or string used for each indent space.

The other thing that this adds is the ability to chain some of the key methods so that it can be combined in single lines of code. the methods that have been modified are:

  • IndentedLoggerAdapter.debug()
  • IndentedLoggerAdapter.info()
  • IndentedLoggerAdapter.warning()
  • IndentedLoggerAdapter.error()
  • IndentedLoggerAdapter.critical()
  • IndentedLoggerAdapter.exception()
  • IndentedLoggerAdapter.log()

po(name=None)¶

Shortcut method for IndentedLoggerAdapter.pop()

pop(name=None)¶

Returns an indent to the top of a FILO queue.

Parameters:name –

This behaives differently depending on what type of data is passed:

int:
If an int is passed, this will pop that many items from the top of the queue.
str:
If a string is passed, this will pop all items from the top of the queue until it reaches that named item. if that item does not exist, it will raise a KeyError.
Returns:Self
pu(*args)¶

Shortcut method for IndentedLoggerAdapter.push()

push(*args)¶

Saves an indent to the top of a FILO queue.

This takes 0-2 arguments, if arguments are passed, the argument that is a string willl be used as the key, the argument that is an int will be used as the indent. if two strings or two int’s are passed, unpredictable results will occur.

Parameters:
  • key (str) – The name to save to, if no string is passed, this will add an un-named item to the queue.
  • indent (int) – The indent to be saved, if None, this saves the current indent.
